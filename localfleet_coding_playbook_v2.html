Claude Code Workflow — For a Non-Coder

🎯 The Core Problem You're Solving

You use AI (Claude Code, Claude.ai) to write your code. The #1 risk is context loss — the AI forgetting what it built, hallucinating function names, creating files that don't connect. This section is your entire defense system. The Pydantic schemas in the Data Contracts tab are the glue that holds everything together across sessions.

The Golden Rules

Rule 1: ONE file per session, TWO files maximum

Each time you open Claude Code, you work on ONE module. Never "build the whole backend." Say "build src/llm/ollama_client.py that takes a string and returns a FleetCommand." Small. Specific. Testable.

Rule 2: Always start every session by pasting the schemas

At the beginning of EVERY Claude Code session, paste the full src/schemas.py from the Data Contracts tab. This is how you keep output consistent across sessions with zero memory between them.

Rule 3: Test before moving on

After Claude Code writes a file, RUN IT in the same session while context is fresh. Don't move to the next module with broken code behind you.

Rule 4: Never let it refactor working code

If a module works, NEVER let Claude Code "improve" it later. It will break things guessing at code it can't fully see. Working code is sacred.

Rule 5: Claude Code for Python, Claude.ai for React

Claude Code is best for Python (can run and test directly). For React components, use Claude.ai — its artifact system lets you preview before committing.

When Claude Code Makes Mistakes

Common AI coding failures — how to catch them
  • Hallucinated imports: References functions that don't exist yet. Fix: check imports against actual files.
  • Wrong field names: Schema says asset_id, it writes vessel_id. Fix: Pydantic catches this at runtime.
  • Ignoring domain types: Treats everything as surface, forgets drone handling. Fix: your session template explicitly lists both domains.
  • Over-engineering: Builds 5 abstract classes when you asked for one function. Fix: say "keep it simple, no abstractions."
  • Breaking working code: Rewrites the whole file when adding a feature. Fix: say "add to existing code, don't change working functions."

Data Contracts — The Source of Truth (v2.0 Multi-Domain)

🔑 Why This Is The Most Important Section

These Pydantic models keep your project consistent across Claude Code sessions. Every module reads and writes these types. The LLM outputs a FleetCommand, the fleet manager inputs a FleetCommand. The simulation outputs AssetState, the WebSocket sends AssetState. ONE schema, EVERYWHERE.

v2.0 changes: Added DomainType enum (SURFACE/AIR), renamed vessel-specific types to asset-generic types, added altitude and GPS-denied fields, added drone-specific patterns.

Create this file FIRST, before any other code. Paste it into every Claude Code session.

File: src/schemas.py

""" LocalFleet Data Contracts v2.0 — Multi-Domain THE source of truth for all modules. Every module imports from this file. DO NOT define data structures elsewhere. v2.0 Changes: - Added DomainType enum (SURFACE, AIR) - Asset-generic naming (AssetCommand, AssetState) - Altitude field for air domain - GPS-denied degradation fields - DronePattern enum for aerial behaviors """ from pydantic import BaseModel, Field from typing import List, Optional from enum import Enum from datetime import datetime # ============================================================ # ENUMS — Shared across all modules # ============================================================ class DomainType(str, Enum): """Which domain this asset operates in.""" SURFACE = "surface" # Vessels — uses CORALL dynamics AIR = "air" # Drones — uses drone_dynamics.py class MissionType(str, Enum): """The 5 mission types LocalFleet supports. No more.""" PATROL = "patrol" # Surface: waypoint circuit SEARCH = "search" # Surface: lawnmower/expanding square ESCORT = "escort" # Surface: protect a reference point LOITER = "loiter" # Surface: hold position AERIAL_RECON = "aerial_recon" # Air: overhead sweep/orbit class FormationType(str, Enum): """How surface vessels arrange relative to each other.""" ECHELON = "echelon" LINE_ABREAST = "line" COLUMN = "column" SPREAD = "spread" INDEPENDENT = "independent" class DronePattern(str, Enum): """How a drone operates over an area.""" ORBIT = "orbit" # Circle a point at altitude SWEEP = "sweep" # Linear pass over area TRACK = "track" # Follow a surface asset STATION = "station" # Hold position at altitude class AssetStatus(str, Enum): """Current operational state of any asset.""" IDLE = "idle" EXECUTING = "executing" AVOIDING = "avoiding" # COLREGS avoidance (surface only) RETURNING = "returning" ERROR = "error" class GpsMode(str, Enum): """GPS availability state.""" FULL = "full" # Normal GPS — precise positions DEGRADED = "degraded" # GPS-denied — noisy positions, reduced rate # ============================================================ # COMMANDS — What the operator sends IN # ============================================================ class Waypoint(BaseModel): """A point in 2D space (meters, local frame).""" x: float y: float class AssetCommand(BaseModel): """ Command for a single asset (surface OR air). The domain field determines which dynamics engine processes it. """ asset_id: str # "alpha", "bravo", "charlie", "eagle-1" domain: DomainType # SURFACE or AIR waypoints: List[Waypoint] speed: float = 5.0 # m/s (surface ~5, drone ~15) altitude: Optional[float] = None # meters — only for AIR domain behavior: str = "waypoint" # "waypoint", "loiter", "search", "orbit", "sweep", "track" drone_pattern: Optional[DronePattern] = None # Only for AIR domain class FleetCommand(BaseModel): """ THE central command object. The LLM produces this from natural language input. Every module that processes commands accepts this. Now supports BOTH surface and air assets in a single command. """ mission_type: MissionType assets: List[AssetCommand] # Can contain mixed SURFACE + AIR formation: FormationType = FormationType.INDEPENDENT # Surface only spacing_meters: float = 200.0 colregs_compliance: bool = True comms_lost_behavior: str = "return_to_base" raw_text: Optional[str] = None # Original natural language command # ============================================================ # STATE — What the simulation sends OUT # ============================================================ class AssetState(BaseModel): """ Current state of one asset (surface or air). Sent via WebSocket to the React dashboard every tick. """ asset_id: str domain: DomainType x: float # Position (meters, local frame) y: float heading: float # Degrees, 0 = North speed: float # m/s altitude: Optional[float] = None # Meters — AIR domain only status: AssetStatus mission_type: Optional[MissionType] = None current_waypoint_index: int = 0 total_waypoints: int = 0 # Surface-specific risk fields risk_level: float = 0.0 # 0.0 = safe, 1.0 = collision imminent cpa: Optional[float] = None # Closest point of approach (meters) tcpa: Optional[float] = None # Time to CPA (seconds) # Drone-specific fields drone_pattern: Optional[DronePattern] = None # GPS degradation gps_mode: GpsMode = GpsMode.FULL position_accuracy: float = 1.0 # Meters of uncertainty (1=precise, 50+=degraded) class FleetState(BaseModel): """ State of the entire fleet. Sent via WebSocket every tick (4Hz). The React dashboard receives this exact JSON shape. """ timestamp: float # Simulation time in seconds assets: List[AssetState] # All surface + air assets active_mission: Optional[MissionType] = None formation: FormationType = FormationType.INDEPENDENT gps_mode: GpsMode = GpsMode.FULL # ============================================================ # EVENTS — What gets logged # ============================================================ class MissionEvent(BaseModel): """A logged event for mission replay.""" timestamp: float event_type: str # "command", "state", "decision", "risk", "gps_change" asset_id: Optional[str] = None domain: Optional[DomainType] = None data: dict created_at: datetime = Field(default_factory=datetime.now) # ============================================================ # API — Request/Response shapes for FastAPI endpoints # ============================================================ class CommandRequest(BaseModel): """POST /api/command — what the dashboard sends.""" text: str # Natural language command source: str = "text" # "text" or "voice" class CommandResponse(BaseModel): """POST /api/command — what the server responds with.""" success: bool fleet_command: Optional[FleetCommand] = None error: Optional[str] = None llm_response_time_ms: Optional[float] = None class GpsDeniedRequest(BaseModel): """POST /api/gps-mode — toggle GPS degradation.""" mode: GpsMode noise_meters: float = 25.0 # Position uncertainty in degraded mode update_rate_hz: float = 1.0 # Reduced from 4Hz in degraded mode

How Modules Connect Through These Schemas

USER (voice/text) │ ▼ CommandRequest { text: "Alpha and Bravo patrol harbor, Eagle aerial recon" } │ ▼ [Fleet Commander] ── calls Ollama with FleetCommand schema ──▶ FleetCommand │ ▼ FleetCommand { mission_type: "patrol", assets: [ { asset_id: "alpha", domain: "surface", waypoints: [...], speed: 5.0 }, { asset_id: "bravo", domain: "surface", waypoints: [...], speed: 5.0 }, { asset_id: "eagle-1", domain: "air", waypoints: [...], altitude: 100, drone_pattern: "sweep" } ], formation: "echelon" } │ ▼ [Fleet Manager] ── routes by domain: ├── SURFACE assets → CORALL dynamics instances └── AIR assets → drone_dynamics.py instance │ ▼ AssetState { asset_id: "alpha", domain: "surface", x: 150, y: 200, heading: 45, ... } AssetState { asset_id: "eagle-1", domain: "air", x: 300, y: 400, altitude: 100, ... } │ ▼ FleetState { timestamp: 12.5, assets: [AssetState, AssetState, ...], gps_mode: "full" } │ ▼ [WebSocket] ── sends JSON to React dashboard every 250ms │ ▼ Dashboard renders vessels (🚢) + drones (✈) on Leaflet map

Anti-Corruption Guide — v2.0

The Build Order (Non-Negotiable)

Build in this EXACT order. Each module depends only on modules above it.

  1. src/schemas.py — Write by hand (copy from Data Contracts tab). Type definitions only.
  2. CORALL modules — Copy from corall-upstream/. Don't modify. Run baseline to verify.
  3. src/dynamics/drone_dynamics.py — Depends on: schemas.py only. Test: can drone follow waypoints?
  4. src/utils/gps_denied.py — Depends on: schemas.py only. Test: does it add noise correctly?
  5. src/llm/ollama_client.py — Depends on: schemas.py + Ollama running. Test: can it parse "patrol harbor with Alpha, Eagle recon" into a multi-domain FleetCommand?
  6. src/fleet/fleet_manager.py — Depends on: schemas.py + CORALL dynamics + drone_dynamics + gps_denied. Test: can 3 vessels + 1 drone track waypoints?
  7. src/fleet/fleet_commander.py — Depends on: schemas.py + ollama_client. Test: NL → FleetCommand → fleet_manager dispatches correctly to both domains?
  8. src/fleet/drone_coordinator.py — Depends on: schemas.py + drone_dynamics. Test: orbit, sweep, track patterns work?
  9. src/api/server.py + ws.py — Depends on: schemas.py + fleet_manager. Test: curl POST /command, see multi-domain state on WebSocket?
  10. dashboard/ — Depends on: WebSocket sending FleetState JSON. Test: map shows vessels AND drone with different icons?
  11. src/voice/whisper_local.py — Depends on: fleet_commander. Test: speak → text → command → both domains respond?
⚠️ If any step breaks

STOP. Fix it before moving forward. A broken module cascades into everything above it.
